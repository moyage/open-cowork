## 1. 为什么建立此变更包

维护者判断：

- `docs/archive/plans/58-v0.2.6-agent-first-dogfood-requirements.md` 是当前 V0.2.6 的主需求基线。
- `docs/archive/reports/2026-04-24-v026-first-step-personal-agent-dogfood-v027-findings.md` 是团队成员第一条自然语言采用指令的 dogfood evidence。
- `docs/archive/plans/59-v0.2.7-first-step-personal-agent-dogfood-requirements.md` 不应拆到未来 V0.2.7，而应并入 V0.2.6 作为团队成员首轮采用验收标准。

原因：V0.2.6 尚未实施。如果把 59 留到 V0.2.7，V0.2.6 可能只修内部一致性，却仍不能让团队成员第一步成功试用。当前版本最需要修复的是 Agent-first adoption 闭环。

## 3. Intent

让新的团队成员在个人域 Agent 中发出第一条自然语言指令后，open-cowork 能完成：

- 安装和配置状态确认；
- 当前项目/当前迭代识别；
- adoption plan 生成；
- source docs 绑定；
- change package 准备；
- 个人域 agent inventory 到 role binding 候选；
- bounded context read set；
- status/current-state/digest 一致；
- 下一步治理动作与人类决策点输出。

## 5. Consolidated requirements

### P0

1. Agent adoption 入口：承接自然语言指令并生成 adoption plan。
2. onboard 真实项目化：不再输出不存在的 `personal-demo`。
3. 状态事实一致：`status/current-state/digest` 对 active change 保持一致。
4. contract/runtime 边界一致：`run` 使用 contract `scope_in/scope_out`。
5. scope validation：检测 `scope_in/scope_out` 冲突。
6. active change 生命周期保护：不静默切换未关闭 change。

### P1

1. bootstrap/update/onboard 输出 Agent-first 状态摘要。
2. diagnose-session recommended read set 进入 handoff pack。
3. 个人域 agent inventory 进入 role binding 候选。
4. source docs 写入 change package。
5. hygiene/doctor 解释 runtime/pending docs/tracked files。
6. evidence doc 支持分析/报告类 artifact。

### P2

1. 生成文档和人类模板默认中文化，保留必要英文锚点。

## 7. Acceptance dogfood

最终验收必须复跑团队成员第一条指令：

```text
根据这个Open-cowork项目（https://github.com/moyage/open-cowork ）的指引，进行安装和配置，在我当前的个人域环境中针对Open-cowork这个项目的迭代实施这套框架的规范和流程
```

验收通过标准：

1. Codex 能生成 adoption plan，而不是手工拼 `pilot` 参数。
2. plan 能引用 58、V0.2.5 dogfood findings、第一条指令 findings、59 作为 source docs。
3. onboard 不再输出不存在的 `personal-demo`。
4. status/current-state/digest 对 active change 保持一致。
5. recommended read set 进入 Agent handoff。
6. 个人域 agent inventory 能进入 role binding 候选。
7. bootstrap 成功反馈不再让人误以为安装失败或需要自己继续敲命令。

## 9. Maintenance note

当前 open-cowork 版本仍存在已知问题：

- `.governance/changes/**` 默认 ignored，运行态 change package 不会自动进入 Git；因此本文作为可追踪基线。
- 当前工具尚未支持 `--source-doc`、`adopt`、hygiene、current-state 自动刷新等能力；这些正是本包要实现的内容。
- V0.2.6 实施前，不应继续把 59 作为 V0.2.7 延后处理。
