# 历史归档

`docs/archive/` 保存 `open-cowork` 迭代过程中的历史计划、设计记录、closeout 和复盘材料。

这些文件用于追溯项目如何演进，不作为新用户默认阅读入口。

## 目录

- `plans/`：历史 milestone 计划、设计包、实施计划和范围切分记录。
- `reports/`：历史状态报告、session recovery playbook、closeout 和复盘材料。

## 阅读建议

- 新用户不要从本目录开始。
- 如果你要理解当前框架，请优先阅读：
  - `../getting-started.md`
  - `../README.md`
  - `../specs/00-top-level-whitepaper.md`
  - `../specs/01-prd.md`
- 如果你要追溯某项能力的设计来源，再进入 `plans/` 或 `reports/`。

## 维护规则

- 新的历史性迭代材料应归档到本目录。
- 当前有效规格应放在 `../specs/`，不要只存在于 archive。
- archive 内部可以保留历史路径和历史表述，但不要再作为 README 的默认阅读路径。
